## 技术铁律(违反即返工)

- **运动学架构:** BikeRoot 锚定、每帧直写 CFrame、扫掠射线碰撞、自管重力。禁止 VehicleSeat / 约束驱动 / Humanoid 载具。(ADR-1)
- **参数只存在于 Config Attributes**(ConfigLive),逻辑代码零魔法数字;配置源码更新用 `.Source` 原地赋值+重 bind(销毁重建会丢用户调参)。
- **约定:** `look=(-sin(yaw),0,-cos(yaw))`,`yaw=atan2(-look.X,-look.Z)`;单位 studs/s、Deg、Sec;`dt=math.min(dt,1/20)`。
- **探针分工:** 中心探针管高度与起飞,前后探针只管姿态法线;改贴地/起飞逻辑前必读 ADR-4,改后必复跑"坡顶起飞回归"。
- **公平:** 计时/奖牌/货币只在服务端产生;敌人与机关固定摆放、确定性刷新;R 重开全部重置。
- **战斗判定:** 局部空间体积查询 + CollectionService Tag 注册的脚本对象;**禁用物理 Touched**(锚定件间不触发)。剑光/子弹均为脚本对象与纯 VFX。(ADR-16)
- **物性声明制:** 可骑面=Terrain+Rideable Tag 白名单,其余 BasePart 一律按障碍走碰撞三档;**碰撞永不产生垂直速度**;重生=烘焙锚点按弧长回取,永不前进。(ADR-27/28)
- **反馈铁律:** gameplay 内反馈永不冻结画面与输入(禁子弹时间/hitstop);慢镜头只进结算页与回放。(ADR-20)
- **物理与视觉分离:** simPos 为物理真相;roll/悬挂等视觉偏移只作用于渲染 CFrame,不得污染 simPos。

## Rojo 工作流

- `default.project.json` = repo↔Studio 映射的唯一事实源;改代码=改 `src/`,经 Rojo 同步,不要在 Studio 里直接改脚本源码。
- **Studio-only 状态**(摩托模型改造、Config Attributes 当前值、控制点、测试台)不在 git 里,以 `docs/status.md`「Studio 侧状态」为准;**代码与 status 两边都更新才算改完**。
- `.rbxl` 定期提交作灾难还原点。

## 工作守则

灰盒优先(V1 禁美术投入)/ 每里程碑独立可玩 + 自动验收(Edit 模式固定 dt=1/60 同步仿真,不需进 Play)/ 人工验收点必停,附《试玩指引:测什么+回答哪几问》/ 破坏性操作先备份到 ServerStorage.NeonRun.Backup / 不确定就问。
